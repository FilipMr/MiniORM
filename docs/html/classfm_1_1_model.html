Detailed Description

Base class for database-backed models (lightweight ORM-style).

A Model represents a SQLite table and a collection of registered columns. Columns (implementing IColumn) typically register themselves into a Model during construction (see fm::Column constructor).

Note
This implementation assumes the first registered column (index 0) is the primary key for operations like find(), update(), and remove().

Loads a row by primary key into this model instance.

Assumes the first registered column is the primary key and is integer-based. Executes:

SELECT * FROM TableName WHERE pk = ?;

On success, each column is populated using IColumn::loadFromSQL().

Registers a column with this model.

Columns call this automatically in their constructor:

owner->register_column(this);
Parameters
colPointer to a column (must remain valid for the Model lifetime).
Warning
The Model does not own the column pointers and does not delete them. Also, registration order matters: columns[0] is treated as primary key.

bool fm::Model::update ( sqlite3 * db)
inline

Updates the row identified by the primary key (first column).

Builds:

UPDATE TableName SET c2=?, c3=?, ... WHERE pk=?;

The primary key column (index 0) is excluded from the SET clause and is bound last in the WHERE clause.

Parameters
dbOpen SQLite database handle.
Returns
true if the UPDATE executed successfully, false otherwise.
Warning
Requires at least one registered column.
Note
This does not check whether any row was actually modified.
